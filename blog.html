Question 1: Discuss the scope of var, let, and const.

ans 1: In JavaScript, var, let, and const are used to declare variables, but they have different scopes and behaviors. Let's discuss the scope of each:
var scope-
If a var variable is declared inside a function, it is accessible anywhere within that function. Variables declared with var are also hoisted to the top of their scope, meaning they are moved to the top of their containing function or the global scope during the compilation phase.
let scope-
They are only accessible within the block they are declared in, whether it's a loop, conditional statement, or a function. let variables are not hoisted to the top of their scope. They are only accessible after the declaration statement.
const scope-
The key difference is that once a value is assigned to a const variable, it cannot be reassigned. However, this doesn't mean the value is immutable. If the value is an object, its properties can still be modified. const variables must be assigned a value when declared; they cannot be left uninitialized.

Question 2: Tell us the use cases of null and undefined.

ans 2: null and undefined are two distinct values in JavaScript that indicate the absence of a meaningful value, but they are used in slightly different contexts. Here are their common use cases:
undefined:
1. Variable Initialization: If you declare a variable but don't assign any value to it, it will hold the value undefined by default.
2. Function Return: If a function does not explicitly return a value, it implicitly returns undefined.

null:
1. Explicit Absence of Value: Unlike undefined, null is typically used to explicitly indicate that a variable or object property has no value or is empty.
2. Comparison for Intentional Absence: In some cases, developers use null as a specific value for comparison or to check for the absence of an expected value.

Question 3: What do you mean by REST API?

ans 3: REST stands for "Representational State Transfer." It's an architectural style and a set of constraints for designing networked applications, especially web services. A RESTful API (Application Programming Interface) is a type of API that adheres to the principles and constraints of the REST architecture.
1. Uniform Interface: REST APIs have a consistent and uniform way of accessing resources (data). This is achieved through standard HTTP methods like GET (retrieve), POST (create), PUT (update), and DELETE (remove).
2. Caching: Responses from a RESTful API can be cached to improve performance and reduce the need for redundant requests.
3. Representation: Data is transferred between the client and server in a specific format, often JSON or XML. This format represents the state of the resource being requested or manipulated.
